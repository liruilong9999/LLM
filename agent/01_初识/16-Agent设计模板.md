# Agent 设计模板

## 本篇目标

本篇提供一份可复制的 Agent(智能体) 设计文档模板。你可以用它来设计天气 Agent、知识库助理、数据分析师、客服系统或内部自动化助手。

## 1. 背景与目标

### 业务背景

写清楚当前业务遇到的问题，不要一开始就写“要做一个 Agent”。

```text
当前问题：
目标用户：
业务影响：
为什么现有方案不够：
```

### Agent 目标

```text
Agent 要完成什么：
不负责什么：
第一版成功标准：
```

示例：

```text
Agent 要完成：回答内部制度问题，并在资料不足时转人工。
不负责：自动审批流程、修改制度文档。
第一版成功标准：常见问题命中率达到 80%，无越权引用。
```

## 2. 用户与场景

| 用户类型 | 典型问题 | 权限范围 | 风险等级 |
| --- | --- | --- | --- |
| 普通员工 | 查询制度 | 公开制度 | 低 |
| 部门主管 | 查询部门流程 | 部门文档 | 中 |
| 管理员 | 管理知识库 | 配置和审计 | 高 |

## 3. 输入与输出

### 输入

```text
文本：
文件：
图片：
音频：
系统事件：
```

### 输出

```text
自然语言回答：
结构化结果：
图表：
工单或草稿：
需要用户确认的动作：
```

## 4. 总体流程

```mermaid
flowchart LR
    用户输入 --> 输入检查
    输入检查 --> 意图识别
    意图识别 --> 任务规划
    任务规划 --> 工具调用
    工具调用 --> 结果检查
    结果检查 --> 回复生成
    回复生成 --> 输出护栏
    输出护栏 --> 用户
```

## 5. 状态设计

```json
{
  "conversation_id": "",
  "user_id": "",
  "goal": "",
  "intent": "",
  "current_step": "",
  "tool_results": [],
  "risk_level": "",
  "need_human": false,
  "final_answer": ""
}
```

字段说明：

| 字段 | 含义 | 是否持久化 |
| --- | --- | --- |
| conversation_id | 会话编号 | 是 |
| user_id | 用户编号 | 是 |
| goal | 当前任务目标 | 是 |
| intent | 意图分类 | 是 |
| current_step | 当前执行步骤 | 是 |
| tool_results | 工具结果摘要 | 视情况 |
| risk_level | 风险等级 | 是 |
| need_human | 是否需要人工介入 | 是 |

## 6. 工具清单

| 工具名 | 类型 | 输入 | 输出 | 权限 | 风险 | 失败策略 |
| --- | --- | --- | --- | --- | --- | --- |
| search_docs | 查询 | query | 文档片段 | doc:read | 低 | 返回未找到 |
| get_order_status | 查询 | order_id | 订单状态 | order:read | 中 | 转人工 |
| create_ticket | 写入 | title、description | 工单编号 | ticket:create | 中 | 用户确认 |


## 7. 记忆策略

| 记忆类型 | 内容 | 写入条件 | 删除策略 |
| --- | --- | --- | --- |
| 短期记忆 | 当前对话 | 自动 | 会话结束清理 |
| 工作记忆 | 当前任务状态 | 自动 | 任务归档 |
| 用户偏好 | 输出格式、语言 | 用户确认 | 用户可删除 |
| 知识库 | 文档片段 | 文档入库 | 跟随源文档版本 |

## 8. 风险与护栏

### 输入护栏

- 用户身份校验。
- 文件类型和大小限制。
- 提示注入检测。
- 高风险意图识别。

### 工具护栏

- 参数 schema(结构约束) 校验。
- 权限检查。
- 超时和重试。
- 敏感操作人工确认。

### 输出护栏

- 敏感信息脱敏。
- 引用检查。
- 结构化输出校验。
- 拒答策略。

## 9. 人工介入

必须转人工的情况：

- 权限不足。
- 工具结果冲突。
- 涉及资金、隐私、法律或生产配置。
- Agent 置信度低。
- 用户明确要求人工。

转人工摘要模板：

```text
用户问题：
已确认信息：
已执行步骤：
工具结果：
当前风险：
建议人工动作：
```

## 10. 评测方案

| 指标 | 目标 | 评测方式 |
| --- | --- | --- |
| 任务成功率 | 80% 以上 | 人工标注样例 |
| 幻觉率 | 越低越好 | 检查无依据结论 |
| 工具成功率 | 95% 以上 | 工具调用日志 |
| 越权率 | 0 | 权限测试 |
| 平均延迟 | 按场景设置 | 日志统计 |
| 单次成本 | 按预算设置 | 模型调用记录 |

评测样例格式：

```json
{
  "id": "case-001",
  "input": "我的退款什么时候到账？",
  "expected_behavior": "查询订单退款状态，并说明预计到账时间",
  "allowed_tools": ["get_order_status"],
  "forbidden_tools": ["issue_refund"],
  "risk_level": "medium"
}
```

## 11. 版本路线

| 版本 | 目标 | 范围 | 验收 |
| --- | --- | --- | --- |
| v0.1 | 跑通原型 | 单工具、单场景 | 能完成演示 |
| v0.2 | 加入记忆 | 会话状态、用户偏好 | 多轮一致 |
| v0.3 | 加入权限 | 工具过滤、资源过滤 | 无越权 |
| v0.4 | 加入评测 | 样例集、回归测试 | 指标稳定 |
| v1.0 | 小流量上线 | 灰度、监控、人工兜底 | 可运营 |

## 12. 设计评审清单

- 是否写清楚不做什么。
- 是否有工具契约。
- 是否有状态字段。
- 是否有失败策略。
- 是否有权限设计。
- 是否有人工介入规则。
- 是否有评测样例。
- 是否有上线和回滚方案。
