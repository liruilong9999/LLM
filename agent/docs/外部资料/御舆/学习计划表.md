# 御舆学习计划表

## 适用对象

这份计划适合已经有编程基础、但第一次系统学习 Agent Harness(智能体运行时骨架) 的读者。目标不是背术语，而是读完后能自己画出一套可落地的 Agent 架构。

## 学习总目标

- 能解释 Claude Code 这类 Agent 为什么比普通聊天机器人更像“可执行的软件系统”。
- 能说清对话循环、工具系统、权限管线、上下文管理、记忆系统、Hook(钩子)、Fork(分叉执行)、MCP(Model Context Protocol，模型上下文协议) 的职责边界。
- 能依据第 15 章写出自己的最小 Agent Harness 设计草图。

## 建议节奏

- 总周期：14 天
- 单次时长：60 到 90 分钟
- 节奏建议：每天读 1 个主题，隔 2 到 3 天做一次小结
- 学习方法：每次都回答 3 个问题
  1. 这一章解决了什么工程问题？
  2. 它引入了什么新模块？
  3. 如果删掉这个模块，系统会出什么问题？

| 周次 | 检查问题 | 达标标准 |

## 推荐做法

- 每读完一章，只记“问题、方案、代价”三件事，不要急着抄太多细节。
- 先盯主链路，再看扩展点，不然很容易在细节里迷路。
- 第 10 天以后开始反复回看 `附录/A-源码导航地图.md`，这会帮助你把零散知识重新接回整体。
- 第 14 天不要只停在阅读，至少写一版你自己的模块划分。
