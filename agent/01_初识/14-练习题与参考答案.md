# 练习题与参考答案

## 本篇目标

本篇用于把前面章节转化成可训练的题目。建议先独立完成，再看参考答案。题目分为基础理解、设计题、场景题和综合项目题。

## 使用方式

1. 先写自己的答案。
2. 对照参考答案检查遗漏。
3. 把做错的题回到对应章节复习。
4. 选择 1 到 2 个综合题写成完整设计文档。

## 一、基础理解题

### 题 1：Agent 和普通聊天机器人有什么区别？

要求：至少从目标、工具、记忆、行动四个角度回答。

参考答案：

普通聊天机器人主要根据当前上下文生成文本；Agent(智能体) 则围绕目标进行规划、调用工具、观察结果、维护状态，并在必要时采取行动。Agent 不是“更长的 prompt(提示词)”，而是一个包含控制器、模型、工具、记忆和治理机制的软件系统。

### 题 2：为什么 LLM 需要工具？

参考答案：

LLM(Large Language Model，大语言模型) 默认不知道实时信息，不能直接访问企业系统，也不能直接执行真实动作。工具让模型能查询天气、读取数据库、搜索文档、创建工单或执行受控代码。工具必须有清晰名称、参数 schema(结构约束)、返回结构、权限和错误处理。

### 题 3：短期记忆、工作记忆、长期记忆有什么区别？

参考答案：

短期记忆是当前会话和最近工具结果；工作记忆是当前任务执行中的结构化状态，例如当前步骤和已完成动作；长期记忆是跨会话保留的信息，例如用户偏好、历史摘要和知识库。三者的生命周期、存储方式和权限策略不同。

### 题 4：MCP 中 tools、resources、prompts 的区别是什么？

参考答案：

tools(工具) 用于执行动作，例如查询订单；resources(资源) 用于提供上下文材料，例如文件或数据源；prompts(提示模板) 用于复用任务说明，例如代码评审模板。判断标准是：是否执行动作、是否只是读取材料、是否是复用流程。

## 二、设计题

### 题 5：设计一个“会议纪要 Agent”

需求：

用户上传会议录音，Agent 生成会议纪要、行动项和风险点。

请写出：

- 输入。
- 工具。
- 记忆。
- 流程。
- 风险控制。

参考答案：

```text
输入：音频文件、会议主题、参会人列表。
工具：ASR(自动语音识别)、说话人分离、文本总结、任务提取、日历或工单创建。
记忆：当前会议文本、历史项目背景、用户纪要格式偏好。
流程：上传音频 -> 转写 -> 按议题分段 -> 提取决策和行动项 -> 生成纪要 -> 用户确认 -> 可选创建任务。
风险控制：低置信度转写标注；行动项创建前确认；敏感会议限制访问；保留音频和文本来源。
```

### 题 6：为“订单查询工具”写工具契约

参考答案：

```yaml
name: get_order_status
description: 查询当前用户有权访问的订单状态、物流状态和退款状态
type: read_only
input_schema:
  order_id:
    type: string
    required: true
    description: 订单编号
output_schema:
  ok:
    type: boolean
  order_id:
    type: string
  status:
    type: string
  logistics_status:
    type: string
  refund_status:
    type: string
  updated_at:
    type: string
permissions:
  - order:read
timeout_ms: 3000
risk_level: low
failure_modes:
  - order_not_found
  - permission_denied
  - upstream_timeout
```

### 题 7：什么时候应该使用 LangGraph？

参考答案：

当任务有多步骤、多分支、人工确认、持久状态、恢复执行、可回放调试等要求时，适合使用 LangGraph(图式智能体编排框架)。如果只是单轮问答或少量工具调用，简单控制器或 LangChain(大语言模型应用开发框架) Agent 就够了。

## 三、场景题

### 题 8：用户问“帮我删除所有测试订单再重新统计销售额”，Agent 应该怎么做？

参考答案：

不应直接执行删除。正确流程：

1. 识别到“删除”是高风险写操作。
2. 检查用户权限。
3. 解释不能直接删除生产数据。
4. 建议先用过滤条件排除测试订单进行统计。
5. 如果确实要删除，生成变更计划并走人工审批。

### 题 9：知识库没有检索到答案，Agent 应该怎么回答？

参考答案：

应该明确说明“当前资料中没有找到依据”，可以提供已检索范围、可能需要补充的资料，以及转人工或继续搜索的选项。不能凭模型常识编造政策或制度。

### 题 10：工具连续两次超时，Agent 应该怎么处理？

参考答案：

应该停止重试，说明工具暂时不可用，并根据场景选择降级方案：请求稍后再试、转人工、使用缓存结果或返回部分结果。需要记录工具失败日志，避免无限循环和成本失控。

## 四、综合项目题

### 题 11：设计一个“私人知识库助理”

要求：

- 支持 Markdown 和 PDF。
- 回答必须引用来源。
- 用户可以删除长期记忆。
- 文档按权限检索。

参考答案要点：

```text
文档接入：解析 Markdown 标题、PDF 页码。
分块策略：按标题或段落分块，保留 source、page、heading、updated_at。
检索策略：权限过滤 -> 关键词检索 -> 向量检索 -> 重排。
回答策略：基于引用片段回答，不足则拒答。
记忆策略：用户偏好经确认后写入，支持删除。
评测：命中率、忠实度、引用准确性、拒答能力、越权检测。
```

### 题 12：设计一个“自动化数据分析师”

要求：

- 用户用自然语言提问。
- 系统只允许只读查询。
- 报告要包含图表和不确定点。

参考答案要点：

```text
数据目录：表、字段、指标口径、权限。
查询生成：自然语言 -> 指标识别 -> SQL 草稿 -> 静态检查 -> 权限检查 -> 执行。
代码解释器：沙箱运行，限制文件、网络、CPU、时间。
图表：按分析目标选择折线图、柱状图、散点图等。
报告：事实、观察、假设、建议分级表达。
安全：禁止写操作，限制大查询，敏感字段脱敏。
```

### 题 13：设计一个“企业客服 Agent”

要求：

- 支持知识库问答、订单查询、工单创建。
- 高风险问题转人工。
- 有 A/B test(对照实验) 和灰度发布方案。

参考答案要点：

```text
核心模块：渠道层、会话管理、意图识别、知识库 Agent、工具 Agent、质量审核、人工客服。
工具权限：订单查询只读，工单创建先生成草稿，补偿和退款需人工审批。
转人工规则：投诉、退款、法律、隐私、工具冲突、低置信度。
评估指标：首响时间、自助解决率、转人工准确率、幻觉率、用户满意度、成本。
灰度发布：内部测试 -> 1% 低风险流量 -> 扩大灰度 -> 全量低风险问题。
```

## 五、评分标准

| 维度 | 低分表现 | 高分表现 |
| --- | --- | --- |
| 概念理解 | 只会背术语 | 能解释为什么这样设计 |
| 工程完整性 | 只有模型和 prompt | 包含工具、状态、权限、评测 |
| 风险意识 | 忽略失败和越权 | 有拒答、人工确认和审计 |
| 可落地性 | 方案空泛 | 有输入输出、流程和验收标准 |
| 可迭代性 | 一次性大而全 | 有版本路线和评测闭环 |
