# 术语表

## 本篇目标

本篇整理 Agent 学习中高频术语，便于回查。首次学习时不必一次背完，建议在阅读其他章节时遇到再回来查。

## Agent 基础术语

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| Agent | 智能体 | 以模型为核心，能规划、调用工具、维护状态并完成目标的软件系统 |
| LLM | 大语言模型 | 根据上下文生成和理解自然语言的模型 |
| prompt | 提示词 | 给模型的任务说明、约束和上下文 |
| system prompt | 系统提示词 | 优先级较高的全局行为规则 |
| context window | 上下文窗口 | 模型单次调用可处理的文本范围 |
| token | 模型处理文本的基本单位 | 成本、长度和延迟通常与 token 数有关 |
| hallucination | 幻觉 | 模型生成无依据或错误内容 |
| grounding | 基于证据生成 | 要求回答建立在工具结果或引用资料上 |

## 规划与推理

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| planning | 任务规划 | 把目标拆成可执行步骤 |
| chain-of-thought | 思维链 | 按步骤推理的方式 |
| tree-of-thought | 思维树 | 同时探索多个候选路径 |
| ReAct | 推理与行动 | 在“思考、调用工具、观察结果”之间循环 |
| workflow | 工作流 | 更固定、可控的步骤编排 |
| state machine | 状态机 | 根据状态和事件决定下一步 |
| stop condition | 停止条件 | 决定 Agent 何时结束或暂停 |

## 记忆与检索

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| short-term memory | 短期记忆 | 当前会话和近期工具结果 |
| working memory | 工作记忆 | 当前任务执行中的结构化状态 |
| long-term memory | 长期记忆 | 跨会话保留的信息 |
| RAG | 检索增强生成 | 先检索资料，再基于资料回答 |
| embedding | 嵌入向量 | 把文本转换成表示语义的数字向量 |
| vector database | 向量数据库 | 用于存储和相似度搜索向量 |
| chunk | 文本块 | 文档切分后的检索单位 |
| rerank | 重排 | 对候选检索结果重新排序 |
| hybrid search | 混合检索 | 结合关键词、向量、元数据过滤等方式 |

## 工具与协议

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| tool | 工具 | Agent 可调用的外部能力 |
| tool calling | 工具调用 | 模型选择工具并生成参数，由程序执行 |
| schema | 结构约束 | 描述输入输出字段、类型和要求 |
| API | 应用程序编程接口 | 系统之间交互的约定 |
| MCP | 模型上下文协议 | AI 应用连接工具、资源和提示模板的开放协议 |
| MCP host | MCP 宿主应用 | 用户直接使用的 AI 应用 |
| MCP client | MCP 客户端 | 宿主应用内负责协议连接的组件 |
| MCP server | MCP 服务器 | 暴露工具、资源和提示模板的服务 |
| resource | 资源 | 可读取的上下文材料 |
| prompt template | 提示模板 | 可复用的任务说明模板 |
| transport | 传输方式 | MCP 消息如何在客户端和服务器间传递 |

## 框架与编排

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| LangChain | 大语言模型应用开发框架 | 提供模型、工具、结构化输出等组件 |
| LangGraph | 图式智能体编排框架 | 用图、节点、边和状态组织复杂 Agent 流程 |
| node | 节点 | 图中的一个处理步骤 |
| edge | 边 | 节点之间的流转关系 |
| checkpoint | 检查点 | 保存中间状态，便于恢复和回放 |
| interrupt | 中断 | 暂停流程，等待人或外部系统补充 |
| middleware | 中间件 | 在模型或工具调用前后插入通用逻辑 |

## 安全与治理

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| guardrail | 护栏 | 对输入、工具和输出的安全控制 |
| sandbox | 沙箱 | 限制代码或工具执行影响范围的环境 |
| least privilege | 最小权限原则 | 只授予完成任务所需的最小权限 |
| audit log | 审计日志 | 记录关键操作，便于追踪和问责 |
| prompt injection | 提示注入 | 外部内容诱导模型忽略原有规则 |
| human-in-the-loop | 人在回路中 | 高风险步骤由人确认或处理 |
| redaction | 脱敏 | 隐藏或删除敏感信息 |
| rate limit | 限流 | 限制调用频率以保护系统 |
| rollback | 回滚 | 出问题时恢复到上一个稳定版本 |

## 评测与上线

| 术语 | 中文解释 | 一句话理解 |
| --- | --- | --- |
| evaluation | 评测 | 用样例和指标衡量 Agent 表现 |
| regression test | 回归测试 | 防止旧问题再次出现的测试 |
| golden dataset | 黄金评测集 | 经过人工确认的高质量测试样例 |
| A/B test | 对照实验 | 比较两个方案哪个效果更好 |
| canary release | 灰度发布 | 先给小部分用户使用，观察稳定后扩大 |
| observability | 可观测性 | 能追踪系统内部运行状态 |
| latency | 延迟 | 从请求到响应的耗时 |
| throughput | 吞吐量 | 单位时间内可处理的请求数量 |
