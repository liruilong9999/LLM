# LLM（Large Language Model，大语言模型）学习路线索引

这份索引不是单纯目录，而是一条建议的学习路径。目标是让你先建立整体地图，再理解核心机制，最后把训练、推理和部署串成一个完整工程闭环。

主文档入口：[LLM从Transformer到训练推理部署](./LLM从Transformer到训练推理部署.md)

## 这套材料会帮你建立什么能力

- 用自己的话解释大语言模型（LLM, Large Language Model）到底在做什么
- 理解 Token（模型处理的最小离散文本单元）、Embedding（嵌入表示）和上下文表示之间的关系
- 区分预训练（Pretraining）、监督微调（SFT, Supervised Fine-Tuning）和对齐训练（Alignment Training，让输出更符合人类偏好）
- 看懂推理（Inference，使用已训练模型生成结果）中 Prefill（首轮全上下文计算）和 Decode（逐 Token 解码）的区别
- 理解量化（Quantization）与 KV Cache（Key-Value Cache，历史注意力键值缓存）为什么会直接影响部署性能

## 推荐阅读顺序

1. 先读主文档第 1 到第 2 章，建立“LLM 为什么能工作”的概念地图
2. 再读第 3 到第 4 章，理解 Token 化、Embedding、训练目标和训练阶段
3. 接着读第 5 到第 6 章，理解推理解码、KV Cache、量化和服务部署
4. 最后读第 7 到第 9 章，消化常见误区、学习路线和术语速查

## 四阶段学习法

### 第 1 阶段：先建立全景图

目标：

- 知道 LLM 是什么，不是什么
- 理解 Transformer（以注意力机制为核心的序列建模架构）为什么取代了很多旧序列模型
- 建立“文本 -> Token -> 向量 -> 概率分布 -> 输出文本”的主线

重点章节：

- 主文档第 1 章“什么是 LLM”
- 主文档第 2 章“为什么主流是 Transformer”
- 主文档第 3 章“文本如何变成 Token 和 Embedding”

完成标准：

- 能解释 Token 为什么不等于字、不等于词
- 能解释 Self-Attention（自注意力，让序列中的位置彼此建立关联）在做什么
- 能解释为什么 Decoder-only（只有解码器堆叠的自回归结构）适合文本生成

阶段产出：

- 用 5 句话写出 LLM 的工作主线
- 自己画一张简化版“文本进入模型”的流程图

### 第 2 阶段：吃透训练主线

目标：

- 理解模型为什么能学会“预测下一个 Token”
- 理解预训练、SFT、对齐训练各自解决什么问题
- 理解训练不只是“喂数据”，而是数据、损失、梯度和优化器共同作用

重点章节：

- 主文档第 4 章“训练：从语料到参数”

完成标准：

- 能解释交叉熵损失（Cross-Entropy Loss，用于衡量预测分布与真实标签差距）在训练中的角色
- 能说出基础模型、指令模型、对齐后模型之间的差别
- 能说明为什么训练成本同时受数据规模、参数规模、序列长度和硬件并行方式影响

阶段产出：

- 写出预训练、SFT、RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）/DPO（Direct Preference Optimization，直接偏好优化）的区别
- 用自己的语言解释“参数为什么会更新”

### 第 3 阶段：吃透推理与部署主线

目标：

- 理解模型推理时为什么要逐 Token 生成
- 理解 Prefill、Decode、采样策略、KV Cache 的作用
- 理解部署时的资源瓶颈和性能指标

重点章节：

- 主文档第 5 章“推理：模型如何一步步生成文本”
- 主文档第 6 章“部署：从模型文件到服务系统”

完成标准：

- 能解释 Greedy Decoding（贪心解码）、Top-k、Top-p 和 Temperature（温度参数，用于调节输出分布平滑度）的区别
- 能说明 TTFT（Time To First Token，首 Token 延迟）和吞吐（单位时间生成 Token 或处理请求的能力）的意义
- 能解释为什么量化通常先用于部署而不是训练主流程

阶段产出：

- 画出“用户请求 -> 推理服务 -> 模型输出”的链路图
- 列出影响推理延迟的 5 个因素

### 第 4 阶段：形成自己的知识结构

目标：

- 把概念、训练、推理、部署连成一张完整脑图
- 能识别常见误区，避免只会背术语
- 能判断自己下一步该深入模型原理、训练方法还是工程系统

重点章节：

- 主文档第 7 章“常见误区”
- 主文档第 8 章“下一步学习建议”
- 主文档第 9 章“术语速查”

完成标准：

- 能独立复述 LLM 全链路，不依赖原文
- 能把新接触的术语挂回已有知识图，而不是孤立记忆
- 能说清自己接下来要补哪一块，为什么

阶段产出：

- 用一页笔记整理自己的 LLM 心智图
- 写出 10 个关键词，并给出一句话定义

## 推荐阅读方法

- 第一次阅读时先抓因果链，不追求一次记住所有缩写
- 第二次阅读时重点盯术语，检查每个专业词是否真的理解而不是眼熟
- 第三次阅读时把文中的图和表重新整理成自己的版本

## 常见阅读误区

- 只看模型结构，不看数据和训练目标
- 只看训练，不看推理和部署
- 把“看懂术语”误认为“看懂系统”
- 把“模型会输出”误认为“模型一定知道事实”

## 读完后你应该具备的能力

- 能和工程师讨论模型训练、推理、部署时的核心权衡
- 能和产品或业务同学解释大模型能力边界
- 能继续学习 MoE（Mixture of Experts，专家混合结构）、长上下文、RAG、Agent 或多模态，而不会迷失在零散术语里
